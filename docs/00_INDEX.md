## Implementation

The canonical implementation of the Eigen Universe is available in `src/eigen/`.

*   **Dynamics (The Law)**: `src/eigen/dynamics/` (Hamiltonian & Mechanics)
*   **Gauge (Fields)**: `src/eigen/gauge/` (Symmetry & Invariance)
*   **Techne (Hardware)**: `src/eigen/techne/` (Sources, Sinks, Operators)

### Requirements (Baseline)
- Python >= 3.14 (explicit baseline; no compatibility shims for older versions)
- Explicit runtime activation is required before composing or executing operators:

```python
from eigen.runtime import activate
activate()  # sets QuantumField as the active field
```

### Operator Algebra (Production Semantics)
- Flow (>>): sequential composition → `Hamiltonian([...])`
- Choice (|): fallback semantics → try left; on error or None, use right
- Ensemble (&): structured concurrency (asyncio.TaskGroup) with proper stream teeing
- Interference (+): additive superposition via `Interference([...])`

### Deployment Readiness vs Roadmap
- Production-ready now:
  - Async pipelines with FileSource/Map/Filter/Batch → CSVSink
  - HTTPPort (requires FastAPI/Uvicorn) to expose pipelines as APIs
    - POST /run streams application/jsonl with one valid JSON object per line when the pipeline returns an async iterator
    - GET /stream serves text/event-stream (SSE) with JSON payloads; x-trace-id echoed in response headers
  - Parallel composition (&) and fallback (|) with robust error handling
- Roadmap (future work and integrations):
  - Distributed execution, observability (OpenTelemetry), advanced connectors (Kafka/S3/Parquet)
  - AI/ML platform features (experiment tracking, model registry, scalable serving)
  - Low-latency trading stack (specialized runtimes and exchange connectivity)
